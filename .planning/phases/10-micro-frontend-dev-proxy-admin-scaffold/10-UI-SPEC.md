## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between nav items |
| sm | 8px | Compact element spacing, avatar margin |
| md | 16px | Default element spacing, nav item padding |
| lg | 24px | Top-nav height padding, section padding |
| xl | 32px | Layout horizontal padding |
| 2xl | 48px | Top-nav total height (48px — touch-friendly) |
| 3xl | 64px | Page content top offset below fixed nav |

Exceptions: Top-nav height is 48px (2xl) — this is the minimum touch target for the full nav bar row. Individual nav link hit targets are 44px tall via `py-3` (12px top + 12px bottom + 20px text = 44px).

Source: 8-point scale standard; 44px touch target from WCAG 2.5.5 (AAA guideline).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(0.98 0.01 90)` — cream/off-white | Page background (`--background`), body fill |
| Secondary (30%) | `oklch(1 0 0)` — pure white | Top-nav bar background, cards (`--card`) |
| Accent (10%) | `oklch(0.55 0.25 280)` — vibrant purple | Active nav link indicator, nav logo color, focus ring |
| Destructive | `oklch(0.6 0.25 25)` — red-orange | Sign-out button hover state (if implemented as destructive action) |

Accent reserved for:
1. Active/current route indicator on nav links (underline or background highlight)
2. Kubeasy logo/brand text in the top-nav left slot
3. Focus ring on all keyboard-navigable elements (`--ring` token)

All other interactive elements (nav links, avatar button) use `--foreground` / `--muted-foreground` at rest.

Source: `packages/ui/src/styles/tokens.css` — tokens consumed directly, no new values introduced.

## Layout Contract

### Top Navigation Bar (`components/top-nav.tsx`)

```
┌─────────────────────────────────────────────────────────┐
│  [Kubeasy logo] (left)    [Challenges] [Users] [Settings]    [Avatar ▾] (right)  │
└─────────────────────────────────────────────────────────┘
  height: 48px, bg: white (--card), border-bottom: 2px solid --border
```

- Logo slot (left): Kubeasy wordmark or icon, links to `/admin/challenges`, accent purple color.
- Nav links (center-right): `NavigationMenu` with links to `/admin/challenges`, `/admin/users`, and a disabled `Settings` placeholder. Active link has accent purple underline (2px, offset 4px). Inactive links use `--muted-foreground`.
- User slot (far right): `DropdownMenu` triggered by `Avatar`. Dropdown contains: user display name (non-interactive label), `Separator`, "Sign out" `Button` (destructive variant).
- Top-nav is `position: sticky; top: 0; z-index: 50` — stays visible during scroll.
- Border: `neo-border` equivalent — `border-bottom: 3px solid oklch(0.15 0 0)` (neo-brutalist style, matching `--border` token).

### Full-screen Loading State

Displayed while `authClient.useSession()` `isPending` is true:

```
┌──────────────────────────┐
│                          │
│    [spinner icon]        │
│    "Loading..."          │  ← 14px label / muted-foreground
│                          │
└──────────────────────────┘
  flex h-screen items-center justify-center, bg: --background
```

Implementation: `<div className="flex h-screen items-center justify-center gap-3 bg-background">` with a `<Loader2 className="animate-spin h-5 w-5 text-muted-foreground" />` icon from lucide-react and a `<span>` label.

### Placeholder Page (`challenges/index.tsx`, `users/index.tsx`)

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | "Sign out" (verb only — destructive, no noun needed in this context) |
| Loading state heading | none — icon-only with spinner conveys state |
| Loading state label | "Loading..." (14px, muted-foreground) |
| Challenges placeholder heading | "Challenges" |
| Challenges placeholder body | "Challenge management is coming in Phase 11." |
| Users placeholder heading | "Users" |
| Users placeholder body | "User management is coming in Phase 11." |
| Settings nav label | "Settings" (disabled — no href, `aria-disabled="true"`) |
| Page title (browser tab) | "Kubeasy Admin" |
| Error state (auth redirect) | No copy — silent redirect via `window.location.href`. No error message shown to end users. |
| Destructive action — sign out | No confirmation required. Sign-out is non-destructive (reversible by logging back in). Immediate action on click. |

Note on destructive actions: The only action in Phase 10 is sign-out via the user dropdown. No confirmation dialog is needed — sign-out is fully reversible. The "Sign out" label uses `Button` with `variant="ghost"` and red text (`text-destructive`) to communicate the consequence without a modal overhead.

## Interaction States

### Nav Links

| State | Visual Treatment |
|-------|-----------------|
| Default | `text-muted-foreground`, no decoration |
| Hover | `text-foreground`, `bg-muted` (subtle) |
| Active (current route) | `text-primary` (accent purple), `border-b-2 border-primary` underline |
| Disabled (Settings) | `text-muted-foreground/50`, cursor not-allowed, `aria-disabled="true"` |
| Focus (keyboard) | `ring-2 ring-ring ring-offset-2` (standard shadcn focus style) |

### Avatar / User Menu

| State | Visual Treatment |
|-------|-----------------|
| Default | Avatar with initials, `border-2 border-border` (neo-brutalist) |
| Hover | `shadow-sm` (2px 2px 0 black offset — neo token) |
| Dropdown open | Avatar border becomes `border-primary` |
| Sign-out item hover | `bg-destructive/10`, `text-destructive` |

### Loading Spinner

- Displayed for the duration `isPending === true`
- Minimum display: no artificial delay — show immediately, hide immediately when resolved
- Animation: `animate-spin` (Tailwind utility, 1s linear infinite)
