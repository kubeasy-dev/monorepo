# Phase 5.2: Missing Dashboard & Profile Features - Context

**Gathered:** 2026-03-19
**Status:** Ready for planning

<domain>
## Phase Boundary

Add the missing UI features that existed in the original Next.js app:
- **Dashboard page**: radar chart ("Skills by Themes") + full recent activity section (4-item preview + grouped-by-month modal)
- **Profile page**: edit name, create/revoke API tokens, email subscription management (Resend), and danger zone (reset progress + delete account)

No new features beyond what was in the original Next.js app — except delete account which is being added now. Backend API for email preferences uses TanStack Start server functions (not new Hono routes).

</domain>

<decisions>
## Implementation Decisions

### Dashboard — Radar Chart
- Library: **recharts** — add to `apps/web` dependencies. Port `DashboardChart` component directly.
- Axis labels: **theme names** (human-readable: "Workloads", "Networking"), not slugs. Requires joining `themeListOptions()` data.
- Empty state: **show empty chart axes** — render the full radar grid with 0% on all axes, giving users a sense of what to unlock. Do not hide the chart or show a placeholder message.
- "Best theme" callout below the chart stays (as in the original — shows best theme name + percentage when > 0%).

### Dashboard — Recent Activity
- **Full fidelity**: 4-item inline preview + "View All Activity" button opens a Dialog with grouped-by-month history.
- History limit: **keep 20 items** (current `/xp/history` limit). Pagination is a future improvement — do not implement it now.
- Port `DashboardRecentGainsClient` component as-is, adapting imports from tRPC to TanStack Query + api-client.

### Profile Page — Layout
- **4 stacked sections** (same as original): Profile Settings → API Tokens → Email Preferences → Danger Zone
- No tabs, no accordion.
- Page prefetches XP and email topics data server-side (route loader), same pattern as other protected pages.

### Profile Page — Edit Name
- Port `ProfileSettings` component.
- API call: `PATCH /api/user/name` via `api.user.updateName()` (already in api-client.ts).
- Success: `sonner` toast "Profile updated successfully!".

### Profile Page — API Tokens
- Port `ProfileApiTokens` component.
- Uses `authClient.apiKey.*` **directly** (Better Auth client) — no new Hono routes needed.
  - `authClient.apiKey.list()` — list tokens
  - `authClient.apiKey.create({ name })` — create token (returns full key once)
  - `authClient.apiKey.delete({ keyId })` — delete token
- Newly created token shown once with copy button + "I've saved my token" dismiss.
- Delete confirmation dialog before removing.
- Analytics: `trackApiTokenCreated()` and `trackApiTokenCopied(name)` client-side events.

### Profile Page — Email Preferences (Subscription Management in Resend)
- **TanStack Start server functions** (`createServerFn`) — NOT new Hono routes. Email preference logic lives in `apps/web`, calls Resend directly (like the original Next.js app did with server components).
- Two server functions:
  - `getEmailTopics(userId)` — reads topics from DB + subscription status from Resend API
  - `updateEmailSubscription(userId, topicId, subscribed)` — updates Resend contact topics
- **Optimistic updates**: toggle responds instantly, rolls back on error. Same UX as original.
- **Graceful degradation**: if no `resendContactId` or Resend not configured, show topics with default opt-in status (no error state).
- Port `ProfileEmailPreferences` component, replacing tRPC mutation/query with TanStack Query + server functions.

### Profile Page — Danger Zone
- Two actions (original had one — adding delete account):
  1. **Reset All Progress** — calls `DELETE /api/user/progress` via `api.user.resetProgress()` (add to api-client). Confirmation dialog required.
  2. **Delete Account** — calls Better Auth's account deletion (research correct Better Auth client method). Confirmation dialog required. Redirect to landing page after deletion.
- Port `ProfileDangerZone` component and extend with delete account section.

### Claude's Discretion
- Better Auth method for account deletion (check Better Auth docs/client)
- Exact `createServerFn` API shape for email preferences
- Error boundary / error state for radar chart if theme data fails to load
- Recharts responsive container sizing

</decisions>
