```caddyfile
# Kubeasy Production Reverse Proxy
# Upstream addresses injected via Railway env vars at runtime.
# Local dev uses Turborepo MFE proxy (microfrontends.json), not this file.

kubeasy.dev {
    auto_https off

        reverse_proxy {$ADMIN_UPSTREAM}
    }

    }
}
```

**1. Create `apps/admin/nginx.conf`** (per D-06, Claude's discretion on exact contents):

nginx config for SPA with `/admin/` base path. CRITICAL: Use `alias` not `root` — Vite builds to `dist/` (no `admin/` subdirectory), but URL paths start with `/admin/`. Using `root` would look for files in `/usr/share/nginx/html/admin/` which does not exist.

```nginx
server {
    listen 80;
    server_name _;

    # Gzip compression for text assets
    gzip on;
    gzip_types text/html text/css application/javascript application/json image/svg+xml;
    gzip_min_length 256;

    # SPA at /admin/ base path
    location /admin/ {
        alias /usr/share/nginx/html/;
        try_files $uri $uri/ /admin/index.html;

        # Cache-busted assets (Vite hashed filenames) — cache aggressively
        location ~* \.(js|css|woff2|png|svg|ico)$ {
            expires 1y;
            add_header Cache-Control "public, immutable";
        }
    }

    # Redirect bare /admin to /admin/
    location = /admin {
        return 301 /admin/;
    }
}
```

**2. Create `apps/admin/Dockerfile`** (per D-06):

Two-stage build: Node.js stage builds the Vite SPA, nginx stage serves it.

```dockerfile
# Stage 1: Build the Vite SPA
FROM node:22-alpine AS builder

RUN corepack enable && corepack prepare pnpm@latest --activate

WORKDIR /app

# Copy workspace config for pnpm install
COPY package.json pnpm-lock.yaml pnpm-workspace.yaml turbo.json ./
COPY packages/ packages/
COPY apps/admin/ apps/admin/

# Install workspace deps needed for admin build
RUN pnpm install --frozen-lockfile

# Build the SPA (tsc --noEmit && vite build)
WORKDIR /app/apps/admin
RUN pnpm build

# Stage 2: Serve with nginx
FROM nginx:alpine

COPY --from=builder /app/apps/admin/dist /usr/share/nginx/html
COPY apps/admin/nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80
```

IMPORTANT: The Docker build context must be the REPO ROOT (not `apps/admin/`), because the build stage needs `packages/`, `pnpm-lock.yaml`, and `pnpm-workspace.yaml` from the root. The `COPY apps/admin/nginx.conf` in stage 2 copies from the build context (repo root), not from the builder stage.

Note on `VITE_API_URL`: This is a Vite build-time env var (prefixed with `VITE_`). It gets compiled into the JavaScript bundle. For production, Railway must set `VITE_API_URL=https://kubeasy.dev` as a build-time env var on the admin service. This is a Railway dashboard configuration step, not a Dockerfile change.
