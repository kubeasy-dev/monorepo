  <action>
**CRITICAL SEQUENCE: OAuth URIs FIRST, then API_URL change, then DNS cutover. If DNS is cut before URIs are registered, all OAuth logins break immediately.**

**Step 1: Add new OAuth redirect URIs (keep old ones during transition)**

1. GitHub OAuth App (github.com > Settings > Developer settings > OAuth Apps):
   - Add callback URL: `https://kubeasy.dev/api/auth/callback/github`
   - Keep existing `https://api.kubeasy.dev/api/auth/callback/github`

2. Google OAuth Console (console.cloud.google.com > APIs and Services > Credentials):
   - Add authorized redirect URI: `https://kubeasy.dev/api/auth/callback/google`
   - Keep existing `https://api.kubeasy.dev/api/auth/callback/google`

3. Microsoft Entra (portal.azure.com > App registrations > Authentication):
   - Add redirect URI: `https://kubeasy.dev/api/auth/callback/microsoft`
   - Keep existing `https://api.kubeasy.dev/api/auth/callback/microsoft`

**Step 2: Update API_URL on API service**
1. Railway dashboard > `api` service > Variables
2. Change `API_URL` from `https://api.kubeasy.dev` to `https://kubeasy.dev`
3. Redeploy API service

**Step 3: DNS cutover**
1. Assign custom domain `kubeasy.dev` to the Caddy Railway service
2. Update DNS provider with Railway CNAME target
3. Keep `api.kubeasy.dev` active as fallback during transition

**Step 4: Production smoke test**
1. `curl https://kubeasy.dev/` — web app loads
2. `curl https://kubeasy.dev/admin/` — admin SPA loads
3. `curl https://kubeasy.dev/api/health` — returns 200
4. Test OAuth with GitHub, Google, Microsoft — full login flows
5. Test SSE: challenge submission shows real-time validation updates
6. Test admin: challenges page + users page load with data

**Step 5: Cleanup (optional, after 24h stable)**
1. Remove old OAuth redirect URIs (`https://api.kubeasy.dev/api/auth/callback/*`)
2. Consider removing public domain from API service
