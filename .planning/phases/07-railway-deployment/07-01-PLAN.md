1. Add `"build": "tsc"` script to `apps/api/package.json` scripts section. This compiles TypeScript from `src/` to `dist/` per tsconfig.json outDir. Place it after the existing `start` script.

2. Create `apps/api/Dockerfile` with the Turborepo 3-stage pattern:

**Stage 1 -- Prepare** (named `prepare`):
```
FROM node:22-alpine AS prepare
RUN apk add --no-cache libc6-compat
RUN corepack enable && corepack prepare pnpm@10.32.1 --activate
RUN npm install -g turbo@2
WORKDIR /app
COPY . .
RUN turbo prune @kubeasy/api --docker
```

**Stage 2 -- Builder** (named `builder`):
```
FROM node:22-alpine AS builder
RUN apk add --no-cache libc6-compat
RUN corepack enable && corepack prepare pnpm@10.32.1 --activate
WORKDIR /app
COPY --from=prepare /app/out/json/ .
COPY --from=prepare /app/out/pnpm-lock.yaml ./pnpm-lock.yaml
COPY --from=prepare /app/out/pnpm-workspace.yaml ./pnpm-workspace.yaml
RUN pnpm install --frozen-lockfile
COPY --from=prepare /app/out/full/ .
RUN pnpm turbo run build --filter=@kubeasy/api
```

**Stage 3 -- Runner** (named `runner`):
```
FROM node:22-alpine AS runner
RUN apk add --no-cache libc6-compat
WORKDIR /app
ENV NODE_ENV=production
RUN addgroup -S appgroup && adduser -S appuser -G appgroup
```

For the runner COPY commands, the JIT packages (@kubeasy/api-schemas, @kubeasy/jobs, @kubeasy/logger) export raw `.ts` that gets compiled by `tsc` in the builder stage. Since `apps/api/tsconfig.json` uses `moduleResolution: "Bundler"` and `module: "ESNext"`, the compiled `dist/` JS files will contain import paths like `@kubeasy/logger` that resolve via node_modules symlinks at runtime. Therefore the runner must include:

```
COPY --from=builder --chown=appuser:appgroup /app/apps/api/dist ./dist
COPY --from=builder --chown=appuser:appgroup /app/apps/api/node_modules ./node_modules
COPY --from=builder --chown=appuser:appgroup /app/apps/api/package.json ./package.json
COPY --from=builder --chown=appuser:appgroup /app/node_modules ./root_node_modules
```

WAIT -- the workspace packages are symlinked in node_modules. The runner needs access to the packages' compiled output OR the raw .ts source. Since there is no compilation of these packages, their `.ts` files must be present. The safest approach:

```
COPY --from=builder --chown=appuser:appgroup /app/apps/api/dist ./apps/api/dist
COPY --from=builder --chown=appuser:appgroup /app/apps/api/package.json ./apps/api/package.json
COPY --from=builder --chown=appuser:appgroup /app/node_modules ./node_modules
COPY --from=builder --chown=appuser:appgroup /app/packages ./packages
```

Then set WORKDIR to /app/apps/api and CMD:
```
USER appuser
WORKDIR /app/apps/api
EXPOSE 3001
CMD ["node", "--import", "./dist/instrumentation.js", "dist/index.js"]
```

IMPORTANT: After building locally with `docker build -f apps/api/Dockerfile -t kubeasy-api .`, inspect the container to verify the workspace package imports resolve correctly. If the build succeeds but container crashes with "Cannot find module '@kubeasy/logger'", the node_modules symlinks may not resolve. In that case, the `COPY` strategy must be adjusted to also copy packages source into the runner.

The approach above copies the full `node_modules` (which contains pnpm symlinks to packages/) and the `packages/` directory. This ensures workspace package resolution works. The image size is larger than ideal but correctness matters more.

**Stage 3 -- Runner** (named `runner`):

The Vinxi/TanStack Start build produces `.output/` which is a self-contained directory. The `.output/server/index.mjs` is the entry point. The `.output/` directory contains all necessary bundled code (Vite bundles everything including node_modules into the output). However, some native modules or large dependencies may still need node_modules.

```
FROM node:22-alpine AS runner
RUN apk add --no-cache libc6-compat
WORKDIR /app
ENV NODE_ENV=production
RUN addgroup -S appgroup && adduser -S appuser -G appgroup
COPY --from=builder --chown=appuser:appgroup /app/apps/web/.output ./.output
USER appuser
EXPOSE 3000
CMD ["node", ".output/server/index.mjs"]
```

NOTE: If the Vinxi `.output/` is NOT fully self-contained and requires node_modules at runtime, the COPY strategy must be expanded to include:
```
COPY --from=builder --chown=appuser:appgroup /app/apps/web/node_modules ./node_modules
COPY --from=builder --chown=appuser:appgroup /app/node_modules ./root_node_modules
```

Build locally first with `docker build -f apps/web/Dockerfile -t kubeasy-web .` and test. If the container crashes with module resolution errors, add node_modules to the runner.

The executor MUST build and test locally. If `.output/server/index.mjs` fails to start because of missing modules, iteratively add the required COPY commands.
