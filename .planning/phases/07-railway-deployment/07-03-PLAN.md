This task requires human action on the Railway dashboard. Claude cannot automate Railway project creation. Present the following instructions to the user:

**Step 1: Create Railway project**
- Go to Railway dashboard, create a new project
- Connect it to the kubeasy/website GitHub repository

**Step 2: Add PostgreSQL plugin**
- In the project, click "New" > "Database" > "PostgreSQL"
- Note: Railway auto-injects `DATABASE_URL` into services that reference it

**Step 3: Add Redis plugin**
- Click "New" > "Database" > "Redis"
- Note: Railway auto-injects `REDIS_URL` into services that reference it
- IMPORTANT: After Redis is provisioned, connect via Railway CLI or the Redis console and run:
  `CONFIG SET maxmemory-policy noeviction`
  Then verify: `CONFIG GET maxmemory-policy` should return `noeviction`

**Step 4: Deploy SigNoz**
- In the same project, deploy SigNoz via the Railway template: https://railway.com/template/signoz (or search "SigNoz" in Railway templates)
- This deploys multiple services: signoz (UI), signoz-otel-collector, clickhouse, zookeeper
- Note the exact name of the otel-collector service (likely `signoz-otel-collector`)
- The internal hostname will be: `http://<service-name>.railway.internal:4318`

**Step 5: Create api service**
- Click "New" > "GitHub Repo" > select the kubeasy/website repo
- Set service variable: `RAILWAY_CONFIG_PATH=apps/api/railway.json`
- Set the following environment variables on the api service:
  - `DATABASE_URL` = `${{Postgres.DATABASE_URL}}` (Railway variable reference)
  - `REDIS_URL` = `${{Redis.REDIS_URL}}` (Railway variable reference)
  - `BETTER_AUTH_URL` = `https://api.kubeasy.dev` (or your production API domain)
  - `BETTER_AUTH_SECRET` = (your secret)
  - `GITHUB_CLIENT_ID` = (from GitHub OAuth app)
  - `GITHUB_CLIENT_SECRET` = (from GitHub OAuth app)
  - `GOOGLE_CLIENT_ID` = (from Google OAuth)
  - `GOOGLE_CLIENT_SECRET` = (from Google OAuth)
  - `MICROSOFT_CLIENT_ID` = (from Microsoft OAuth)
  - `MICROSOFT_CLIENT_SECRET` = (from Microsoft OAuth)
  - `RESEND_API_KEY` = (from Resend dashboard)
  - `OTEL_EXPORTER_OTLP_ENDPOINT` = `http://signoz-otel-collector.railway.internal:4318` (adjust service name if different)
  - `OTEL_SERVICE_NAME` = `kubeasy-api`
  - `NODE_ENV` = `production`
- Assign a custom domain: `api.kubeasy.dev` (or your domain)

**Step 6: Create web service**
- Click "New" > "GitHub Repo" > select the same repo
- Set service variable: `RAILWAY_CONFIG_PATH=apps/web/railway.json`
- Set the following environment variables on the web service:
  - `VITE_API_URL` = `https://api.kubeasy.dev` (the api service public URL)
  - `OTEL_EXPORTER_OTLP_ENDPOINT` = `http://signoz-otel-collector.railway.internal:4318`
  - `OTEL_SERVICE_NAME` = `kubeasy-web`
  - `NODE_ENV` = `production`
  - `NOTION_TOKEN` = (Notion API token for blog content)
  - `NOTION_DATABASE_ID` = (Notion database ID)
  - Any other web-specific env vars from your .env
- Assign a custom domain: `kubeasy.dev` (or your domain)

**Step 7: Trigger deployment**
- Both services should auto-deploy from the main branch
- Watch build logs: confirm both use Dockerfile builder (not Nixpacks/Railpack)
- Confirm both builds succeed

Wait for user to type "deployed" or describe issues.
