Create `railway.json` config-as-code files for both `apps/api` and `apps/web` Railway services, specifying Dockerfile builder, watch patterns, start commands, and health checks.

Purpose: DEPLOY-02 requires per-service watch paths so that a change to `apps/web` only rebuilds web, and a change to `packages/` rebuilds both. The `railway.json` files are checked into git and referenced via `RAILWAY_CONFIG_PATH` service variable on Railway.

Output: Two `railway.json` files ready for Railway deployment.

<!-- Railway config-as-code schema (from Railway docs) -->

railway.json schema fields used:
- build.builder: "DOCKERFILE" | "NIXPACKS" | "RAILPACK"
- build.dockerfilePath: string (absolute from repo root)
- build.watchPatterns: string[] (glob patterns, absolute from repo root)
- deploy.startCommand: string (overrides Dockerfile CMD)
- deploy.healthcheckPath: string (HTTP path for health probe)
- deploy.healthcheckTimeout: number (seconds, default 300)
- deploy.restartPolicyType: "ON_FAILURE" | "ALWAYS" | "NEVER"
- deploy.restartPolicyMaxRetries: number

From apps/api/src/routes/index.ts:
```typescript
routes.get("/health", (c) => c.json({ status: "ok" }));
```
Health endpoint is at /api/health (routes are mounted at /api prefix).

IMPORTANT: railway.json does NOT follow Root Directory. Each service on Railway must set
RAILWAY_CONFIG_PATH=apps/api/railway.json (or apps/web/railway.json) as a service variable.
</interfaces>

1. Create `apps/api/railway.json` with the following exact content:

```json
{
  "$schema": "https://railway.com/railway.schema.json",
  "build": {
    "builder": "DOCKERFILE",
    "dockerfilePath": "apps/api/Dockerfile",
    "watchPatterns": [
      "apps/api/**",
      "packages/**"
    ]
  },
  "deploy": {
    "startCommand": "node --import ./dist/instrumentation.js dist/index.js",
    "healthcheckPath": "/api/health",
    "healthcheckTimeout": 300,
    "restartPolicyType": "ON_FAILURE",
    "restartPolicyMaxRetries": 3
  }
}
```

Key decisions:
- `dockerfilePath` is absolute from repo root (not relative to railway.json location)
- `watchPatterns` includes both `apps/api/**` and `packages/**` per CONTEXT.md decision
- `startCommand` replicates the exact start script from package.json, including `--import` flag for OTel
- `healthcheckPath` is `/api/health` which returns `{ status: "ok" }` (confirmed in codebase)
- `restartPolicyType: "ON_FAILURE"` with 3 max retries provides crash recovery

2. Create `apps/web/railway.json` with the following exact content:

```json
{

}
```

Key decisions:

- `healthcheckPath` is `/` (the landing page, which is pre-rendered and always available)
- Same restart policy as api
