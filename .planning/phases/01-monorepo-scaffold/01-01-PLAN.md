**1. Create `turbo.json` at repo root** with this exact content:
```json
{
  "$schema": "https://turborepo.com/schema.json",
  "ui": "tui",
  "envMode": "loose",
  "globalEnv": ["DATABASE_URL", "REDIS_URL", "BETTER_AUTH_SECRET"],
  "tasks": {
    "build": {
      "dependsOn": ["^build"],
      "outputs": [".next/**", "!.next/cache/**", "dist/**"]
    },
    "typecheck": {
      "dependsOn": ["^typecheck"]
    },
    "dev": {
      "cache": false,
      "persistent": true
    },
    "lint": {
      "dependsOn": ["^lint"]
    }
  }
}
```

Key points:
- `tasks` key (NOT `pipeline` — Turborepo v2)
- `envMode: "loose"` for Phase 1 per user decision
- `globalEnv` includes `DATABASE_URL`, `REDIS_URL`, `BETTER_AUTH_SECRET` for cache key inputs
- `dependsOn: ["^build"]` ensures packages compile before apps (INFRA-03)

**2. Update `pnpm-workspace.yaml`** to add workspace packages discovery while preserving existing content:
```yaml
packages:
  - '.'          # root (.) included as workspace member per locked decision
  - 'packages/*'
  - 'apps/*'

onlyBuiltDependencies:
  - '@tailwindcss/oxide'
  - esbuild
  - sharp
  - unrs-resolver
```

Note on root workspace membership: the locked decision specifies adding `.` to the packages list. The entry above satisfies this literally. pnpm also auto-includes root without it, but the explicit `.` keeps the intent visible in the generated file.
