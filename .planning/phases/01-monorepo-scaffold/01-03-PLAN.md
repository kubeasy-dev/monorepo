**2. Create `docker/otel-collector-config.yaml`:**
```yaml
receivers:
  otlp:
    protocols:
      grpc:
        endpoint: 0.0.0.0:4317
      http:
        endpoint: 0.0.0.0:4318

processors:
  batch:

exporters:
  debug:
    verbosity: detailed

extensions:
  zpages:
    endpoint: 0.0.0.0:55679

service:
  extensions: [zpages]
  pipelines:
    traces:
      receivers: [otlp]
      processors: [batch]
      exporters: [debug]
    metrics:
      receivers: [otlp]
      processors: [batch]
      exporters: [debug]
    logs:
      receivers: [otlp]
      processors: [batch]
      exporters: [debug]
```

Key: `zpages` is listed under BOTH `extensions:` block AND `service.extensions` list. The exporter is `debug` only (stdout) — no real backend in Phase 1.

**3. Create `docker-compose.yml` at repo root:**
```yaml
services:
  postgres:
    image: postgres:17-alpine
    environment:
      POSTGRES_DB: kubeasy
      POSTGRES_USER: kubeasy
      POSTGRES_PASSWORD: kubeasy
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U kubeasy"]
      interval: 5s
      timeout: 5s
      retries: 5

  redis:
    image: redis:7-alpine
    command: redis-server --maxmemory-policy noeviction
    ports:
      - "6379:6379"
    volumes:
      - redis_data:/data
    healthcheck:
      test: ["CMD", "redis-cli", "ping"]
      interval: 5s
      timeout: 5s
      retries: 5

  otel-collector:
    image: otel/opentelemetry-collector-contrib:0.123.0
    command: ["--config=/etc/otel-collector-config.yaml"]
    volumes:
      - ./docker/otel-collector-config.yaml:/etc/otel-collector-config.yaml
    ports:
      - "4317:4317"    # gRPC OTLP
      - "4318:4318"    # HTTP OTLP
      - "55679:55679"  # zpages debug UI
    depends_on:
      - redis

volumes:
  postgres_data:
  redis_data:
```

Key points:
- PostgreSQL 17 Alpine with credentials `kubeasy/kubeasy/kubeasy` (db/user/password)
- Redis 7 Alpine with `--maxmemory-policy noeviction` (required for BullMQ reliability per REAL-04)
- OTel Collector uses `contrib` image (not base) for zpages support
- Named volumes `postgres_data` and `redis_data` persist across restarts
- Health checks on postgres and redis for container readiness
- OTel Collector ports: 4317 (gRPC), 4318 (HTTP), 55679 (zpages debug UI)
- `otel-collector` depends_on `redis` (not strictly necessary but orders startup)
